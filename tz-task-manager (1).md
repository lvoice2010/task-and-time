# ТЗ: Таск-менеджер «Дела» — канбан + тайм-трекер

## 1. Общее описание

Веб-приложение для персонального управления задачами и учёта рабочего времени. Совмещает канбан-доску (матрица Эйзенхауэра) с тайм-трекером и аналитикой по направлениям и компаниям.

**Целевой пользователь:** руководитель, работающий одновременно на несколько компаний в разных функциональных направлениях. Необходимо видеть, сколько часов уходит на каждое направление и каждую компанию, чтобы управлять балансом.

**Правило:** 40 рабочих часов в неделю. Минимум 2 часа из 8 — продажи, минимум 1 час из 8 — маркетинг.

---

## 2. Технический стек

- **Frontend:** React (JSX), один файл-артефакт
- **Стилизация:** inline styles + CSS-in-JS (без внешних CSS-файлов)
- **Шрифты:** DM Sans (UI), JetBrains Mono (цифры, таймеры, теги)
- **Хранение данных:** `window.storage` API (persistent storage между сессиями)
- **Ключ хранения:** `kanban-v4`
- **Тема:** тёмная (фон `#0B0E14`)

---

## 3. Структура данных

### 3.1 Задача (Task)

```typescript
interface Task {
  id: string;              // случайный ID (9 символов, base36)
  title: string;           // название задачи
  column: ColumnId;        // текущая колонка
  dept: DeptId | null;     // направление (отдел)
  company: CompanyId | null; // компания
  sessions: Session[];     // массив рабочих сессий
  createdAt: number;       // timestamp создания
}

interface Session {
  start: number;   // timestamp начала
  end: number | null; // timestamp окончания (null = активная сессия)
}
```

## 4. Функциональные требования

### 4.1 Вкладка «Канбан» (основная)

#### Канбан-доска
- 6 колонок, горизонтальный скролл при необходимости
- Drag-and-drop задач между колонками
- Счётчик задач в заголовке каждой колонки
- Суммарное время (Σ) по колонке

#### Карточка задачи
- Теги: статус-бейдж + тег отдела + тег компании
- Название задачи (клик → inline-редактирование)
- Таймер с тремя кнопками:
  - **▶ Старт** — начать работу (создаёт новую сессию)
  - **⏸ Пауза** — приостановить (закрывает текущую сессию, время сохраняется)
  - **⏹ Стоп** — завершить работу (аналогично паузе, семантически)
- Если задача ранее была на паузе, кнопка показывает «▶ Прод.» (продолжить)
- Текущее накопленное время на карточке
- Индикатор активной задачи (пульсирующая зелёная точка)
- Кнопка ◆ — циклическое переключение отдела (null → sales → marketing → ops → null)
- Кнопка ● — циклическое переключение компании (null → kg → kc → pf → null)
- Кнопка × — удаление задачи (с подтверждением нет, мгновенно)
- Дата создания

#### Статусы задачи
| Статус | Бейдж | Условие |
|---|---|---|
| Ожидает | серый | нет сессий |
| В работе | зелёный | есть активная (незакрытая) сессия |
| Пауза | жёлтый | есть закрытые сессии, но нет активной |

#### Ограничение: одновременно активна только одна задача
- При нажатии «Старт» на новой задаче — предыдущая автоматически ставится на паузу
- Текущая активная задача отображается в Summary Bar

#### Форма добавления задачи
- Поле ввода названия
- Чипы выбора отдела (Продажи / Маркетинг / Операционка) — опционально
- Чипы выбора компании (КГ / КЦ / ПФ) — опционально
- Enter — добавить, Escape — отменить

#### Фильтры
- Панель фильтрации по отделу и компании (кнопки-переключатели)
- Можно комбинировать (пересечение)
- Счётчик в колонке показывает `filtered/total`
- Если все задачи скрыты фильтром — текст «N скрыто»

#### Summary Bar (сводка)
- Общее количество задач
- Количество «Сделано»
- Суммарное время (все задачи)
- Суммарное время «Сделано»
- Текущая активная задача (название)

### 4.2 Вкладка «Отчёты»

#### Выбор периода
- Кнопки: Сегодня, 7 дней, 30 дней, Всё время, Свой период
- Свой период: два `<input type="date">` (от → до)
- Сессии, пересекающие границу периода, обрезаются корректно (эффективный старт/конец)

#### Общая сводка
- Общее время за период (крупно, в часах)
- Количество задач с трекингом за период

#### Графики «По направлениям» и «По компаниям»
- Горизонтальные бар-чарты
- Каждый бар: название + полоска + значение в часах
- Цвета соответствуют тегам
- Общий масштаб для обоих графиков (одинаковый maxVal)
- Отдельная строка «Без отдела» / «Без компании» для задач без тегов

#### Кросс-таблица «Направление × Компания»
- Строки: направления
- Столбцы: компании
- Значения: часы (с одним знаком после запятой)
- Итоги по строкам и столбцам (Σ)
- Пустые ячейки: «—»

#### Топ задач по времени
- До 15 задач, отсортированных по убыванию времени
- Каждая строка: номер, название, теги, часы
- Только задачи с ненулевым временем за выбранный период

## 5. Логика тайм-трекинга

### Принцип работы
- Время хранится НЕ как единое число, а как массив сессий `{start, end}`
- Это позволяет точно фильтровать по периодам в отчётах
- Активная сессия: `end === null`, при вычислении используется `Date.now()`

### Таймер (визуальный)
- Тикает каждую секунду (setInterval)
- `elapsed` = `Date.now() - timerStart.current` (в секундах)
- При монтировании: если в storage есть `activeTimerId` и `timerStartedAt`, восстановить таймер

### Формат отображения
- `< 1 мин` → `Nс`
- `1–59 мин` → `Nм Nс`
- `≥ 1 ч` → `Nч Nм`
- В отчётах: часы с одним десятичным знаком (`N.N ч`)

## 6. Сохранение данных

### Структура в storage

```json
{
  "tasks": [...],
  "activeTimerId": "abc123" | null,
  "timerStartedAt": 1712345678000 | null
}
```

### Ключ: `kanban-v4`

### Поведение
- Загрузка при монтировании (`useEffect`, async)
- Сохранение при каждом изменении `tasks` или `activeTimerId`
- Кнопка «Сбросить» — `confirm()` → очистка state + удаление из storage

## 7. UI / UX требования

### Тема
- Фон: `#0B0E14`
- Текст: `#E2E8F0` (основной), `#CBD5E1` (заголовки), `#64748B` / `#475569` / `#374151` (вторичный)
- Границы: `rgba(255,255,255, 0.04–0.12)`
- Карточки: полупрозрачный фон `rgba(255,255,255, 0.025)`

### Анимации
- Пульсация зелёной точки у активной задачи (`@keyframes pulse`)
- Hover-эффекты на карточках и кнопках (плавные, `transition: all 0.15–0.2s`)

### Адаптивность
- Горизонтальный скролл доски на малых экранах
- Фильтры и сводка: `flex-wrap: wrap`
- Минимальная ширина колонки: 270px

### Шрифты
- UI: `DM Sans` (400, 500, 600, 700)
- Цифры, таймеры, теги, код: `JetBrains Mono` (400, 500, 600)
- Загрузка через Google Fonts (внутри `<style>` через `@import`)

## 8. Навигация

- Переключатель «Канбан» / «Отчёты» — в шапке, стилизованный как segmented control
- Активная вкладка: голубой фон + текст `#38BDF8`
- Неактивная: прозрачный фон, текст `#64748B`

---

## 9. Кнопка «Сбросить»

- Расположение: правый край шапки
- Стиль: красная обводка, красный текст
- Действие: `confirm("Удалить все задачи и данные?")` → очистка

---

## 10. Известные ограничения текущей реализации

- Нет undo/redo
- Нет экспорта отчётов (CSV/PDF)
- Нет уведомлений/напоминаний
- Нет многопользовательского режима
- Drag-and-drop — нативный HTML5 (не работает на мобильных touch-устройствах)
- Данные хранятся в `window.storage` (привязаны к артефакту Claude)
