# Доработка: модуль «12-недельный план»

## Контекст

В приложении «Дела» (канбан + тайм-трекер) уже есть две вкладки: Канбан и Отчёты. Нужно добавить третью вкладку — **«12 недель»**.

Методология: книга «12 Week Year» (Брайан Моран). Суть — ставишь цели на 12 недель, разбиваешь на ежедневные тактики, отмечаешь выполнение каждый день.

---

## Что добавить

### Новая вкладка «12 недель» в навигации

Добавить третью кнопку в существующий segmented control в шапке: **Канбан** | **Отчёты** | **12 недель**

## Структура данных

### План (WeekPlan)

```typescript
interface WeekPlan {
  id: string;
  title: string;                // "Q2 2026" или произвольное название
  startDate: string;            // ISO date начала периода, например "2026-02-22"
  endDate: string;              // автоматически = startDate + 83 дня (12 недель)
  goals: Goal[];
}
```

### Цель (Goal)

```typescript
interface Goal {
  id: string;
  number: number;               // порядковый номер (1, 2, 3...)
  title: string;                // "Похудение до 59 кг"
  company: CompanyId | null;    // привязка к компании (kg/kc/pf) или null (личная)
  startValue: string;           // начальное значение: "67 кг", "120 000 ₽", "0 клиентов"
  targetValue: string;          // целевое значение: "59 кг", "138 000 ₽", "5 клиентов"
  currentValue: string;         // текущее значение (обновляется вручную)
  tactics: Tactic[];
}
```

### Тактика (Tactic)

```typescript
interface Tactic {
  id: string;
  title: string;                // "Зарядка", "1 л воды", "3 ч — продажи"
  frequency: 'daily' | 'weekday';  // ежедневно (7/7) или по будням (5/7)
  completions: Record<string, boolean>;  // "YYYY-MM-DD" → true/false
}
```

## UI: основной экран вкладки

### Шапка плана

- Название плана (редактируемое)
- Период: «22 фев — 16 мая 2026» (автоматически)
- Текущая неделя: «Неделя 6 из 12»
- Кнопка «Настроить план» → открывает режим редактирования
- Если планов несколько — выпадающий список для переключения + кнопка «Новый план»

### Блок прогресса по целям

Под шапкой — горизонтальные карточки целей с прогрессом:

```
┌─────────────────────────────────────────────────────┐
│  1. Похудение до 59 кг                              │
│  Старт: 67 кг  →  Цель: 59 кг  →  Сейчас: 64 кг   │
│  ████████████░░░░░░░░░  37.5%                       │
│  [Обновить]                                         │
├─────────────────────────────────────────────────────┤
│  2. ПФ — увеличение дохода на 15%        ПФ         │
│  Старт: 120 000 ₽  →  Цель: 138 000 ₽  →  Сейчас: 128 000 ₽ │
│  ████████████████░░░░░  57%                         │
│  [Обновить]                                         │
└─────────────────────────────────────────────────────┘
```

- **Прогресс-бар** — вычисляется автоматически, если startValue, targetValue и currentValue содержат числа. Формула: `(current - start) / (target - start) * 100%`
- Если значения нечисловые — прогресс-бар не показывается, только текстовые значения
- Кнопка **«Обновить»** — открывает поле ввода нового currentValue
- Тег компании справа (если привязана)

### Таблица ежедневного трекинга

Сетка как в Excel-скриншоте:

**Строки:** тактики, сгруппированные по целям

**Столбцы:** дни (84 дня = 12 недель × 7)

**Sticky-колонки (не скроллятся):**
- № цели
- Название цели (только в первой строке группы, остальные — пусто)
- Название тактики

**Скроллируемая часть:**
- Заголовки: месяцы сверху, числа дней под ними
- Недели визуально разделены (чередующийся фон или вертикальная линия)

**Ячейка (день × тактика):**
- Размер: ~28×28px
- Клик переключает по циклу: пусто → ✓ (зелёная) → ✗ (красная, явный пропуск) → пусто
- Будущие дни — неактивные (серый фон, некликабельные)
- Текущий день — выделен (вертикальная подсветка столбца, например жёлтая рамка)
- Выходные (сб, вс) — чуть темнее фон
- Для тактик с frequency=weekday: ячейки выходных заблокированы (серые, некликабельные)

**Итоги по неделям (строка внизу каждой недели или под таблицей):**
- % выполнения за неделю = (выполнено / ожидаемо) × 100%
- Ожидаемо: daily = 7, weekday = 5 (для каждой тактики, суммируется)
- Цветовая индикация:
  - ≥ 85% → зелёный
  - 60–84% → жёлтый
  - < 60% → красный

## UI: режим настройки плана

Открывается по кнопке «Настроить план». Можно сделать как:
- модальное окно, или
- замену основного контента (с кнопкой «Назад к трекеру»)

### Поля плана:
- Название плана (текст)
- Дата начала (date picker) → дата окончания считается автоматически (+83 дня)

### Управление целями:
- Список целей с drag-and-drop для сортировки (или кнопки ↑↓)
- Для каждой цели:
  - Название (текст)
  - Компания (select: КГ/КЦ/ПФ/нет)
  - Начальное значение (текст, свободный формат: "67 кг", "120 000 ₽", "0")
  - Целевое значение (текст)
  - Список тактик:
    - Название (текст)
    - Частота (radio: ежедневно / по будням)
    - Кнопка удалить
  - Кнопка «+ Добавить тактику»
- Кнопка «+ Добавить цель»
- Кнопка «Удалить цель» (с confirm)
